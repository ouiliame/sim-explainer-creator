# browser-agent demo-corpus

No invented demo content ships in this video — every captured value is
⟨pending⟩ a real run artifact and renders as skeleton lines (see
`artifact-request.md`, written per protocol and stored UNSENT — batch
build, no live workspace).

| File | What it is |
| --- | --- |
| `artifact-request.md` | The unsent request + the ⟨pending⟩ value table |
| `grounding.md` | Product-truth sources for everything that IS on screen |

The demo flow the video shows (Start {Input: Competitor} → Research agent
with Exa/Firecrawl/Browser Use → Response {"brief": <research.content>})
is assembled from registry-authored templates and the docs'
BUILD_AGENT_WORKFLOW pattern; the per-value provenance lives in
`grounding.md`.
