# demo-corpus — agent-economy

- `grounding-v1.md` — provenance for every on-screen value: product-source
  truths (URL shape, client list, tool naming, colors, glyphs) and the
  invented demo content (the Scout workflow, fictional companies).
- `artifact-request.md` — the live-workspace request, written but UNSENT
  (batch mode). Run-record values stay off screen until it lands.

Demo flows supported: the Scout competitor-research workflow's editor run,
six MCP-client calls (one per client badge + bookend), and the outbound
`pricing_intel` tool call to a partner workspace's server.
