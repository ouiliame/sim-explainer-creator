# Intro — "What am I looking at?"

Conceptual spine for the Intro Video Sequence. These are the **animated concept
beats** that wrap around the screen-recorded product demos (asking Mothership,
running a workflow, reading logs). Remotion owns the diagrams; the live product
is captured separately and intercut.

Source outline: `curriculum/Intro Video Sequence.md`.

## Locked scene list

1. **what-is-sim** (5s) — Workspace frame appears. One line: "a workbench for building AI systems." Sets the macro object.
2. **object-model** (9s) — The core objects fade into the workspace as tiles: Mothership, Workflow, Table, Knowledge Base, File, Agent, Tool, Logs, Deployment. The keystone vocabulary beat.
3. **system-combines** (8s) — A real system lights up across the tiles: a Workflow reads a Knowledge Base, writes a Table, saves a File, emits Logs. Connector lines trace the flow. "Most systems combine several of these."
4. **exploration-loop** (6s) — The closing loop: Ask → Inspect → Run → Logs → Adjust, drawn as a cycle. "Build something small, inspect what happens."

Total ≈ 28s @ 60fps.

## Narration (reference)

> Sim is a workbench for building AI systems.
> A workspace holds the things your system uses — workflows, tables, knowledge bases, files, tools, logs, deployments — and Mothership helps you build them in natural language.
> Most systems combine several of these. A workflow might read a knowledge base, write results to a table, save a file, and leave logs you can inspect.
> The fastest way to learn is to build something small and watch what happens: ask, inspect, run, check the logs, adjust.
