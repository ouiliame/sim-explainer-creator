# Module 5 — Agents v3 (capability-first cut) — feat/module-agents

Registered as `module-5-agents-v3` alongside v2. Talking-head hook stitched
in front; this piece opens self-contained: *"You can build agents in Sim by
creating a workflow. Let's take a normal workflow and change it to use the
Agent block…"*

**The one idea:** drop an Agent block into your workflow and now it can
*think* — and thinking is concrete: it makes an LLM call, like spawning a
fresh chat with ChatGPT or Claude inside your workflow. The Messages param
IS that chat's messages. NOT problem→solution: the keyword router is
scenery (orientation from the known), not an argument; no bluntness pair,
no re-test pair, no who-decides. Capability, then its anatomy.

**Three runs total.** One quick run in scene 1 (establishes what running
looks like), ONE continuous run spanning scenes 2→3 (starts at the morph,
freezes at the live ring, the aside opens inside that frozen moment, and
the same run completes into Slack), and the closing green run.

**Every value from the real beaming-polaris run** (`demo-corpus/
triage-run.md`): toolset Docs/CRM/Search, the charged-twice message, the
toolCalls with real durations, tokens 5074/431. Slack log duration stands
in until the Slack-wired re-run; KB "0 results" is honest, swap after the
seeded re-run.

## Locked scene list (~60s)

1. **a-normal-workflow** (~7s) — Start → Route → Slack assembles; one quick
   run crosses it ("I want a refund" → `billing` resolves in the Message
   row, reverts). Scenery: orients from the known.
   *Beat intent: a normal workflow, working normally.*
2. **add-the-agent** (~7s) — Route morphs into Triage (the event); its
   three tools grow in (the agent arrives configured). A run starts: "I was
   charged twice" resolves into Start, the pulse reaches the block, the
   live ring comes on — and the moment **holds** through the cut.
   *Beat intent: you add this block and now it can think. (What's it
   doing?)*
3. **inside-the-agent** (~24s — the centerpiece) — We open inside the held
   moment. A fresh chat spawns beside the dimmed chain; the annotation beam
   links the Messages row to the first bubble; `<start.message>` resolves
   to the real message in the row and the bubble TOGETHER (the "that's what
   the Messages param means" beat, with dwell time). Thinking →
   customerLookup (CRM chip rings in sync; the real ZenGraph result) →
   Knowledge (Docs rings; "0 results") → the reply. Panel closes, the ring
   releases, and the SAME run completes: pulse to Slack, `<triage.content>`
   resolves to the agent's words. Everything reverts; settle.
   *Beat intent: thinking = an LLM call — a fresh chat with your prompt,
   your data substituted in, your tools on the table. VO anchor: "like
   spawning a new chat with ChatGPT or Claude, right inside your
   workflow."*
4. **the-run-record** (~8s) — The output bundle with the real numbers: logs
   (Start 32ms · Triage 12.2s · Slack), `content`, `tokens` 5074/431, and
   `toolCalls` expanded, key glowing. VO hangs "tool calling" terminology
   here.
   *Beat intent: nothing about the run is a mystery — it's all recorded.*
5. **whats-ahead** (~8s) — The "Refunds" workflow chip grows onto the tool
   line; the Skills row appears. Names only.
   *Beat intent: you can hand it more — procedures, playbooks. Later.*
6. **still-a-workflow** (~6s) — One last run walks the chain green; camera
   eases back. Hold.
   *Beat intent: after everything — still just a workflow.*

## Continuity contract

- One chain, module-1 geometry, throughout. The chat panel (sc. 3) and run
  record (sc. 4) are overlays over the unmoved chain.
- The scene 2→3 cut carries LIVE state: input resolved, ring on, tag
  unresolved — the aside resolves the tag in sync with the bubble. Verified
  pixel-identical like every other boundary.
- Route→Triage morph keeps equal row counts; tools grow in AFTER the morph
  (width-growth chips, exact line heights) so nothing pops.
- All resolutions revert before scenes 3/1 end; boundary frames show
  template values everywhere except the deliberate held state at 2→3.
- Product state language only. Chat/record contents are the real run's
  data, never captions.
